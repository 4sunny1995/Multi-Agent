# 📖 Glossary: Từ điển thuật ngữ chung

Để tránh nhầm lẫn giữa các Agent, các thuật ngữ dưới đây được định nghĩa thống nhất:

| Thuật ngữ | Ý nghĩa trong hệ thống |
| :--- | :--- |
| **BRD** | Business Requirement Document (Tài liệu nghiệp vụ gốc). |
| **Edge Case** | Các trường hợp biên, rủi ro, ngoại lệ cần xử lý. |
| **TDD** | Test-Driven Development (Viết test trước khi viết code). |
| **Red-Green-Refactor** | Chu trình: Test hỏng -> Code cho Pass -> Tái cấu trúc. |
| **Loose Coupling** | Khớp nối lỏng (Giảm sự phụ thuộc giữa các module). |
| **Gatekeeper** | Vai trò của LEADER trong việc chặn/duyệt code. |
| **Turbo** | Các bước thực thi tự động không cần hỏi ý kiến User. |
| **Acceptance Criteria** | Điều kiện nghiệm thu (Given-When-Then). |
| **Root Cause** | Nguyên nhân gốc rễ (Không phải triệu chứng bên ngoài). |

> [!TIP]
> Hãy sử dụng các thuật ngữ này trong báo cáo `walkthrough.md` để đảm bảo tính chuyên nghiệp.
